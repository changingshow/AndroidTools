# 如何使用 AI 从零开发一个 Android App

## 前言

本文记录了一次完整的实践：在没有 Android Studio、没有 Android 开发经验的情况下，仅通过与 AI 对话，完成了一个 Android 应用的开发、打包、签名和发布。

**最终成果**：一款名为「权限监控」的 App，可以查看手机上所有应用的权限使用情况。

---

## 一、明确你的需求

### 1.1 告诉 AI 你想做什么

不需要懂技术细节，用自然语言描述即可：

> "我想做一个安卓软件，功能是查看第三方软件使用系统权限的情况"

### 1.2 让 AI 帮你分析可行性

AI 会帮你：
- 分析需求是否可实现
- 列出需要用到的技术和 API
- 指出可能的限制和问题
- 给出大致的开发计划和时间估计

### 1.3 确认后开始

看完 AI 的分析，觉得可行就让它开始。不用担心技术细节，AI 会处理。

---

## 二、开发过程

### 2.1 AI 会自动完成的事情

- 创建项目结构和所有配置文件
- 编写全部代码
- 设计应用图标
- 配置主题和颜色

### 2.2 你需要做的事情

**几乎什么都不用做**，只需要：

1. **回答 AI 的问题**（如果有的话）
2. **测试并反馈问题**
3. **提出修改意见**

### 2.3 实际对话示例

整个开发过程就是不断对话：

| 你说 | AI 做 |
|-----|------|
| "我想做一个查看应用权限的软件" | 分析需求，制定计划 |
| "好的，开始吧" | 创建完整项目代码 |
| "我没有 Android Studio" | 配置 GitHub Actions 云端打包 |
| "安装后列表是空的" | 修复权限问题，添加刷新机制 |
| "界面不好看" | 重新设计 UI，优化图标 |
| "顶部按钮重复了" | 移除多余按钮，添加关于页面 |

## 三、打包发布（无需 Android Studio）

### 3.1 使用 GitHub Actions

这是本次开发的关键发现：**不需要在本地安装任何 Android 开发工具**。

**原理**：把代码推送到 GitHub，GitHub 的服务器会自动帮你编译打包。

### 3.2 具体步骤

1. **创建 GitHub 账号**（如果没有）
   - 访问 github.com 注册

2. **创建仓库**
   - 点击 New repository
   - 填写仓库名，选择 Public（免费使用 Actions）

3. **推送代码**
   - AI 会帮你执行 git 命令
   - 或者你手动把文件上传到 GitHub

4. **等待自动构建**
   - 进入仓库 → Actions 标签
   - 等待绿色 ✓ 出现（约 3-5 分钟）

5. **下载 APK**
   - 点击完成的 workflow
   - 在 Artifacts 区域下载
   - 解压后得到 APK 文件

### 3.3 安装到手机

- 把 APK 传到手机
- 点击安装（可能需要允许"未知来源"）
- 完成！

## 四、签名问题

### 4.1 为什么需要签名？

Android 系统要求所有应用必须签名才能安装。这是为了：
- 确认应用来源
- 防止应用被篡改
- 允许应用更新

### 4.2 两种 APK 的区别

| 类型 | Debug APK | Release APK |
|-----|-----------|-------------|
| 签名 | 自动签名 | 需要自己的签名 |
| 体积 | 较大 | 较小（压缩优化） |
| 用途 | 测试 | 正式发布 |
| 能否安装 | ✅ 可以 | ❌ 未签名无法安装 |

### 4.3 生成签名密钥

告诉 AI "帮我生成签名密钥"，它会：
1. 使用 keytool 命令生成密钥文件
2. 配置项目使用这个密钥
3. 推送后 Release APK 就能正常安装了

### 4.4 重要提醒

**签名密钥一定要保存好！**

- 丢失密钥 = 无法更新已发布的应用
- 建议备份到安全的地方
- 记住密码

## 五、测试与迭代

### 5.1 发现问题很正常

第一版肯定有问题，这是正常的。本次开发遇到的问题：

| 问题 | 解决 |
|-----|------|
| 构建脚本错误 | AI 修复脚本 |
| 图标不兼容旧版本 | AI 调整资源文件 |
| 代码编译错误 | AI 修复兼容性问题 |
| 权限授权后不刷新 | AI 添加生命周期监听 |
| 界面不好看 | AI 重新设计 UI |

### 5.2 如何反馈问题

直接把错误信息或问题描述发给 AI：

- 构建失败：复制错误日志
- 安装失败：描述现象
- 功能问题：说明期望 vs 实际
- 体验问题：说明哪里不满意

### 5.3 迭代改进

每次修改后：
1. AI 修改代码并推送
2. GitHub Actions 自动重新构建
3. 下载新的 APK 测试
4. 继续反馈问题

通常 3-5 轮迭代就能得到满意的结果。

## 六、经验总结

### 6.1 AI 能做什么

✅ 可以做：
- 分析需求和可行性
- 设计项目结构
- 编写全部代码
- 配置构建和签名
- 设计 UI 和图标
- 修复各种问题
- 解释技术原理

### 6.2 AI 不能做什么

❌ 需要你做：
- 提出需求和想法
- 测试应用功能
- 反馈问题和意见
- 做最终决定

### 6.3 提高效率的技巧

1. **需求要清晰**
   - 好："我想做一个查看应用权限的软件"
   - 差："帮我做个 App"

2. **问题要具体**
   - 好："安装后点击允许权限，列表还是空的"
   - 差："不能用"

3. **反馈要及时**
   - 发现问题马上说
   - 不要积累一堆问题一起说

4. **信任 AI 的判断**
   - 技术细节让 AI 决定
   - 你专注于功能和体验

### 6.4 适合的项目类型

这种方式特别适合：
- 工具类应用
- 个人使用的小工具
- 原型验证
- 学习 Android 开发

不太适合：
- 大型商业项目
- 需要复杂后端的应用
- 对性能要求极高的应用
